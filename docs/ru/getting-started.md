# Начало работы

В этом разделе описано, как установить Relix, пройти аутентификацию и выбрать проект для управления релизами.

## Требования

| Компонент | Минимальная версия | Описание |
|-----------|-------------------|----------|
| **Go** | 1.25+ | Язык программирования и сборочная система |
| **Git** | любая актуальная | Должен быть доступен в `$PATH` |
| **GitLab PAT** | -- | Персональный токен доступа с правами `api` |

Токен можно создать в настройках GitLab: **Settings → Access Tokens → Personal Access Tokens**. Убедитесь, что выбрана область видимости `api` -- она необходима для работы с Merge Request'ами и пайплайнами.

## Установка

Relix собирается из исходного кода:

```bash
git clone https://github.com/miraxsage/relix.git
cd relix
go build -o relix .
```

После сборки рекомендуется переместить бинарный файл в директорию из `$PATH`, чтобы вызывать его из любого каталога:

```bash
mv relix /usr/local/bin/
```

## Первый запуск

Запустите Relix из директории вашего проекта или укажите путь явно:

```bash
relix                         # текущая директория
relix -d /path/to/project     # указать директорию проекта
```

### Флаги CLI

| Флаг | Описание |
|------|----------|
| `-d <путь>` | Указать рабочую директорию проекта |
| `-h`, `--help` | Показать справку |
| `-v`, `--version` | Показать версию приложения |

## Аутентификация

При первом запуске Relix отобразит форму для ввода учётных данных GitLab.

<img width="800" height="auto" alt="Начальная форма аутентификации" src="../screens/auth.png" />

Необходимо указать:

- **GitLab URL** -- адрес вашего GitLab-инстанса (например, `https://gitlab.example.com`)
- **Email** -- email учётной записи GitLab
- **Token** -- персональный токен доступа с правами `api`

<img width="800" height="auto" alt="Заполненная форма аутентификации" src="../screens/auth-filled.png" />

После успешной аутентификации учётные данные сохраняются в системном хранилище ключей (macOS Keychain, GNOME Keyring и т.д.) и не хранятся в виде открытого текста.

## Выбор проекта

После аутентификации Relix загружает список доступных проектов из GitLab.

<img width="800" height="auto" alt="Загрузка списка проектов" src="../screens/project-loading.png" />

Вы увидите список проектов с возможностью поиска по названию. Начните вводить текст, чтобы отфильтровать результаты, и выберите нужный проект.

<img width="800" height="auto" alt="Список проектов с поиском" src="../screens/project-select.png" />

Выбранный проект запоминается в конфигурации и используется при следующих запусках. Сменить проект можно в любой момент через командное меню (`/`).

## Дальнейшие шаги

После аутентификации и выбора проекта вы попадёте на главный экран Relix. Оттуда можно начать создание релиза, просмотреть историю или изменить настройки.

Подробности о процессе релиза и всех доступных действиях описаны в [Руководстве по использованию](usage.md).

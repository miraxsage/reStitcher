# Конфигурация

Relix хранит настройки в файле `~/.relix/config.json`. Конфигурация создаётся автоматически при первом запуске со значениями по умолчанию и обновляется при изменении настроек через интерфейс.

## Файл конфигурации

Путь: `~/.relix/config.json`

Пример структуры:

```json
{
  "selected_project_id": 42,
  "selected_project_path": "group/my-project",
  "selected_project_name": "Group / My Project",
  "selected_project_short_name": "My Project",
  "base_branch": "root",
  "environments": [
    { "name": "develop", "branch_name": "develop" },
    { "name": "test", "branch_name": "testing" },
    { "name": "stage", "branch_name": "stable" },
    { "name": "prod", "branch_name": "master" }
  ],
  "exclude_patterns": ".gitlab-ci.yml\nsprite.gen.ts",
  "pipeline_jobs_regex": "^(build|deploy).*",
  "selected_theme": "indigo",
  "themes": [
    {
      "name": "indigo",
      "accent": "#5F5FDF",
      "foreground": "#D7D7FF",
      "notion": "#5F5F8A",
      "success": "#00D588",
      "warning": "#FFD600",
      "error": "#FF84A8"
    }
  ]
}
```

## Интерфейс настроек

Настройки доступны через командное меню (`/` → Settings) или клавишу на главном экране. Окно настроек содержит две вкладки.

### Вкладка Release

Управляет параметрами процесса релиза:

- **Базовая ветка** -- основная ветка проекта, от которой создаются релизные ветки (по умолчанию `root`)
- **Окружения** -- список целевых окружений с соответствующими git-ветками
- **Исключения файлов** -- паттерны файлов, исключаемых при переносе содержимого
- **Regex пайплайнов** -- регулярное выражение для фильтрации отслеживаемых джобов пайплайна

<img width="800" height="auto" alt="Настройки: вкладка Release" src="../screens/settings-release.png" />

### Вкладка Theme

Управляет визуальным оформлением:

- **Выбор темы** -- переключение между доступными темами
- **Превью цветов** -- наглядный предпросмотр всех семантических цветов текущей темы

## Окружения

Окружения определяют целевые ветки для релизов. Каждое окружение состоит из отображаемого имени и соответствующей git-ветки.

| Параметр | Описание |
|----------|----------|
| `name` | Отображаемое имя (показывается в верхнем регистре) |
| `branch_name` | Имя git-ветки окружения |

Значения по умолчанию:

| Окружение | Ветка |
|-----------|-------|
| DEVELOP | `develop` |
| TEST | `testing` |
| STAGE | `stable` |
| PROD | `master` |

Окружения можно добавлять, удалять и переименовывать через интерфейс настроек.

## Базовая ветка

Базовая ветка (`base_branch`) -- это корневая ветка проекта, от которой ответвляются релизные ветки. По умолчанию используется `root`. При включённом root merge релизная ветка мержится обратно в эту ветку после создания MR.

## Исключение файлов

Поле `exclude_patterns` содержит список паттернов (по одному на строку), определяющих файлы, которые не будут перенесены из исходной ветки в ветку окружения при выполнении релиза.

Правила валидации паттернов:

- Каждый паттерн -- отдельная строка
- Поддерживаются точные имена файлов (`sprite.gen.ts`) и пути (`src/generated/file.ts`)
- Пустые строки игнорируются
- Паттерны применяются при выполнении шага Copy Content

Пример:

```
.gitlab-ci.yml
sprite.gen.ts
docs/internal
```

## Regex пайплайнов

Поле `pipeline_jobs_regex` задаёт регулярное выражение для фильтрации джобов пайплайна, за которыми ведётся наблюдение. Если поле пустое, отслеживаются все джобы. Например, `^(build|deploy).*` будет отслеживать только джобы, начинающиеся с `build` или `deploy`.

## Темы

Relix поддерживает полную настройку цветовой схемы. Темы хранятся в массиве `themes` конфигурационного файла.

### Обязательные поля

| Поле | Описание |
|------|----------|
| `name` | Уникальное имя темы |
| `accent` | Основной акцентный цвет (границы, активные элементы, выделение) |
| `foreground` | Цвет основного текста |
| `notion` | Цвет вспомогательных элементов (подсказки, неактивные вкладки, описания) |
| `success` | Цвет успешных операций |
| `warning` | Цвет предупреждений и отмеченных элементов |
| `error` | Цвет ошибок и критических состояний |

### Опциональные поля

| Поле | Описание | Значение по умолчанию |
|------|----------|-----------------------|
| `background` | Фон приложения | Прозрачный (фон терминала) |
| `accent_foreground` | Текст на акцентном фоне | `foreground` или тема по умолчанию |
| `notion_foreground` | Текст на фоне notion | `foreground` или тема по умолчанию |
| `success_foreground` | Текст на фоне success | `foreground` или тема по умолчанию |
| `warning_foreground` | Текст на фоне warning | `foreground` или тема по умолчанию |
| `error_foreground` | Текст на фоне error | `foreground` или тема по умолчанию |
| `muted` | Приглушённый фон (неактивные кнопки, блоки кода) | Цвет из темы по умолчанию |
| `muted_foreground` | Текст на приглушённом фоне | `foreground` или тема по умолчанию |
| `env_develop` | Цвет окружения DEVELOP | `accent` |
| `env_test` | Цвет окружения TEST | `warning` |
| `env_stage` | Цвет окружения STAGE | `success` |
| `env_prod` | Цвет окружения PROD | `error` |

Все цвета задаются в формате `#RRGGBB`. Для фона можно использовать значение `"transparent"`, чтобы использовать фон терминала.

### Пример пользовательской темы

```json
{
  "name": "ocean",
  "background": "#0D1117",
  "accent": "#58A6FF",
  "accent_foreground": "#FFFFFF",
  "foreground": "#C9D1D9",
  "notion": "#484F58",
  "success": "#3FB950",
  "warning": "#D29922",
  "error": "#F85149",
  "muted": "#161B22",
  "muted_foreground": "#6E7681",
  "env_develop": "#58A6FF",
  "env_test": "#D29922",
  "env_stage": "#3FB950",
  "env_prod": "#F85149"
}
```

Для опциональных полей, которые не указаны явно, применяется трёхуровневая система фоллбэков:
1. Значение конкретного поля (например, `accent_foreground`)
2. Общее значение `foreground` из текущей темы
3. Значение из темы по умолчанию (indigo)

## Учётные данные

Учётные данные GitLab (URL, email, токен) хранятся в системном хранилище ключей операционной системы:

| ОС | Хранилище |
|----|-----------|
| macOS | Keychain |
| Linux | GNOME Keyring / KWallet |

Данные никогда не сохраняются в виде открытого текста. Для изменения учётных данных используйте командное меню (`/` → Logout).

## Расположение файлов

| Файл | Путь | Описание |
|------|------|----------|
| Конфигурация | `~/.relix/config.json` | Настройки приложения и выбранный проект |
| Состояние релиза | `~/.relix/release.json` | Состояние незавершённого релиза (удаляется по завершении) |
| Индекс истории | `~/.local/.relix/releases/index.json` | Список всех релизов |
| Детали релиза | `~/.local/.relix/releases/{timestamp}.json` | Полные данные отдельного релиза |
| Учётные данные | Системный keyring | GitLab URL, email, токен |
